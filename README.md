# 实验室样本流转登记工具

本地桌面版实验室样本全生命周期流转管理系统，支持从样本接收到分装、复核、退回的完整流程追溯。

## 功能特性

### 核心流程
- **样本接收**：创建批次、录入唯一编号、登记数量和来源
- **分装管理**：对已接收样本进行分装操作
- **提交复核**：分装完成后提交给复核员审核
- **复核通过（交接登记）**：复核员审核通过，自动记录交接人（复核员）和交接时间，样本进入可交接状态
- **退回处理**：任意环节发现问题可退回，并记录退回原因
- **撤销退回**：退回后可撤销最近一次状态变更，**保留完整退回历史记录**

### 业务规则
- **编号唯一性**：同一批次内样本编号不可重复，重复保存会被拒绝
- **权限控制**：普通操作员不能直接执行复核通过操作
- **状态追溯**：每次操作记录操作人、时间、原因和备注
- **数据持久化**：所有数据本地存储，程序重启后数据完整保留

## 技术栈

- **Electron** - 桌面应用框架
- **React 18** - UI 框架
- **TypeScript** - 类型安全
- **Vite** - 构建工具
- **本地 JSON 文件** - 数据持久化存储

## 快速开始

### 安装依赖

```bash
npm install
```

### 开发运行

#### 方式一：Electron 桌面版（推荐，数据持久化到本地 JSON 文件）

```bash
npm run dev
```

此命令同时启动 Vite 开发服务器和 Electron 窗口。数据存储在系统用户数据目录的 JSON 文件中，关闭重启后数据完整保留。

#### 方式二：浏览器预览（仅用于界面调试，数据存 localStorage）

```bash
npx vite
```

然后在浏览器中打开终端提示的地址（默认 `http://localhost:5173`）。

> ⚠️ 浏览器模式的数据仅存于浏览器 localStorage，清除浏览器数据会丢失记录。正式使用请用 Electron 桌面版。

### 构建打包

```bash
npm run build
```

构建产物输出到 `dist/` 目录。Electron 生产模式（`app.isPackaged === true`）会从 `dist/index.html` 加载页面。

## 默认账户

系统内置两个测试账户，可在左下角切换：

| 用户名 | 角色 | 权限 |
|--------|------|------|
| 操作员小王 | 普通操作员 | 接收、分装、提交复核、退回 |
| 复核员老李 | 复核员 | 拥有全部权限，含复核通过 |

## 验证步骤

### 场景一：完整正向流程（收样 → 分装 → 复核 → 导出）

1. **创建批次**
   - 打开「样本接收」页面
   - 点击右上角「+ 新建批次」
   - 输入批次编号：`BATCH-20240616-001`
   - 输入批次名称：`6月16日第一批送检`
   - 点击「创建」

2. **接收样本**
   - 在批次下拉框中选择刚创建的批次
   - 输入样本编号：`SAMPLE-001`
   - 输入数量：`5`
   - 输入来源：`内科病房`
   - 点击「确认接收」
   - 下方列表显示已接收的样本

3. **分装操作**
   - 切换到「样本列表」页面
   - 找到刚才的样本，状态为「已接收」
   - 点击「分装」按钮
   - 填写备注：`分装为5管`
   - 点击「确认」
   - 样本状态变更为「已分装」

4. **提交复核**
   - 点击「提交复核」按钮
   - 确认操作
   - 样本状态变更为「待复核」

5. **复核通过**
   - 在左下角用户切换处，切换到「复核员老李」
   - 找到待复核的样本
   - 点击「复核通过」按钮
   - 确认操作
   - 样本状态变更为「已复核通过」

6. **导出交接清单**
   - 切换到「交接导出」页面
   - 选择对应批次
   - 点击「导出 CSV」
   - 选择保存位置，保存文件
   - 用 Excel 打开导出文件，验证内容正确

### 场景二：重复编号校验（失败链路）

1. 选择一个已有样本的批次
2. 尝试录入与已有样本相同的编号
3. 系统提示「同一批次中已存在相同的样本编号，无法保存」
4. 验证：修改编号为新值后可正常保存

### 场景三：权限控制（失败链路）

1. 确保当前用户为「操作员小王」（普通操作员）
2. 将一个样本推进到「待复核」状态
3. 尝试点击「复核通过」
4. 验证：按钮不会出现（普通操作员看不到复核通过按钮）
5. 切换到「复核员老李」
6. 验证：可以看到「复核通过」按钮并执行操作

### 场景四：退回与撤销（失败链路，重点验证历史不丢失）

1. 选择一个「已分装」状态的样本
2. 点击「退回」按钮
3. 填写退回原因：`样本破损`
4. 填写备注：`第3管有裂痕`
5. 点击「确认」
6. 样本状态变更为「已退回」
7. 点击「历史」按钮，查看完整流转记录，应包含：
   - 样本接收
   - 分装
   - 退回（含原因「样本破损」、备注「第3管有裂痕」）
8. **记录当前历史条数（应为 3 条）**
9. 关闭历史弹窗，点击「撤销退回」按钮
10. 样本状态恢复为「已分装」
11. 再次点击「历史」按钮，验证：
    - ⚠️ 历史条数应 **增加 1 条**（原 3 条 → 现 4 条），**不能比之前少**
    - ⚠️ 「退回」记录仍然存在，**原因和备注完整保留**（不被删除）
    - ⚠️ 最后新增一条「撤销退回」记录，引用了原退回原因
    - 状态正确从「已退回」回到「已分装」

### 场景五：数据持久化验证（需在 Electron 桌面版中执行）

1. 确认使用 `npm run dev` 启动 Electron 桌面版（非浏览器）
2. 完成上述操作后，关闭 Electron 窗口
3. 重新执行 `npm run dev` 启动
4. 验证：
   - 批次信息完整保留
   - 所有样本状态与关闭前一致
   - 历史记录完整可查
   - 交接人、交接时间、数量、退回原因、备注等信息准确
   - 已复核通过样本的交接人/交接时间在「交接导出」页正常展示

### 场景六：历史追溯验证

1. 对任意样本执行多次状态变更
2. 点击「历史」按钮
3. 验证每条记录包含：
   - 操作时间
   - 操作人
   - 操作动作
   - 状态变化（从 xx 到 xx）
   - 原因和备注（如有）

1. 打开「样本接收」页面，创建一个新批次
2. 点击「📥 CSV 批量导入」
3. 准备包含 5 条记录的 CSV，其中：
   - 第 1、2、4 行：样本编号唯一且格式正确（应成功）

4. 上传 CSV，点击「预检」
5. 验证预检结果显示：3 条成功、2 条失败，每行标注具体错误原因
6. 点击「确认导入」
7. 导入完成后，点击左侧菜单「📋 导入历史」

9. 点击该记录卡片展开明细
10. 验证逐条明细：
    - 成功行显示绿色「成功」标签
    - 失败行显示红色「失败」标签及具体错误原因

### 场景八：导入历史持久化验证（需在 Electron 桌面版中执行）

## 状态流转图

```
已接收 → 已分装 → 待复核 → 已复核通过（可导出）
  ↓        ↓        ↓
  ←退回←  ←退回←  ←退回←
           ↓
        已退回 → 可撤销回上一状态
```

## 导入任务状态流转

```
draft（草稿） → prevalidated（已预检） → importing（导入中） → completed（已完成）
                                                              ↓
                                                          reverted（已撤销）
                  ↓
              cancelled（已取消）
```
